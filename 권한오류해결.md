# 액세스 거부 오류 해결 방법

## ⚠️ 문제

"액세스가 거부되었습니다" (Access denied) 오류
- TempProject 폴더에 접근할 권한이 없습니다
- Android Studio나 다른 프로세스가 파일을 사용 중일 수 있습니다

## ✅ 해결 방법

### 방법 1: Android Studio 완전 종료 후 삭제 (권장)

1. **Android Studio 완전 종료**
   - 모든 Android Studio 창 닫기
   - 작업 관리자에서 `studio64.exe` 프로세스 확인 및 종료

2. **파일 탐색기에서 삭제**
   - `D:\GlowCam\idol-filter-app\TempProject` 폴더 우클릭
   - 삭제 선택
   - 권한 요청 시 관리자 권한 부여

3. **또는 명령 프롬프트에서 (관리자 권한)**
   ```bash
   # 관리자 권한으로 명령 프롬프트 실행
   cd /d D:\GlowCam\idol-filter-app
   rmdir /S /Q TempProject
   ```

### 방법 2: 올바른 프로젝트 열기 (가장 중요!)

**TempProject는 무시하고 올바른 프로젝트를 열어야 합니다:**

1. **Android Studio 실행**
2. **File → Open**
3. **`D:\GlowCam\idol-filter-app\android` 폴더 선택**
   - ⚠️ **TempProject가 아닌 idol-filter-app\android를 선택!**
4. **OK 클릭**

### 방법 3: TempProject 무시

TempProject는 더 이상 필요 없습니다:
- Android Studio에서 `D:\GlowCam\idol-filter-app\android` 프로젝트만 열기
- TempProject는 그대로 두어도 됩니다 (사용하지 않음)

---

## 🎯 핵심 해결책

**가장 중요한 것: 올바른 프로젝트를 열어야 합니다!**

- ✅ **올바른 프로젝트:** `D:\GlowCam\idol-filter-app\android`
- ❌ **잘못된 프로젝트:** `D:\GlowCam\idol-filter-app\TempProject` (무시)

---

## 📝 확인 사항

1. Android Studio에서 어떤 프로젝트가 열려 있는지 확인
2. `D:\GlowCam\idol-filter-app\android` 폴더를 열었는지 확인
3. TempProject는 사용하지 않으므로 무시해도 됩니다

TempProject 폴더를 삭제할 수 없다면:
- 그대로 두고 무시
- 올바른 프로젝트(`idol-filter-app\android`)만 사용
- 나중에 컴퓨터 재시작 후 삭제
